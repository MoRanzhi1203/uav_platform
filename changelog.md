# 无人机群林农协同系统 - 变更说明

## 本次更新概览
本次更新对地形管理模块进行了深度重构与规范化，重点解决了地块类型定义模糊、子类型管理交互不统一、以及默认配置缺失等问题。通过本次改动，系统地块属性编辑功能已达到生产级可用标准。

## 主要变更点

### 1. 地块类型规范化
- **移除“混合”类型**：在 `TerrainArea` 和 `TerrainZone` 模型中移除了 `mixed` 选项，统一地块分类为：林区 (`forest`)、农田 (`farmland`)、建筑 (`building`)、水域 (`water`)、道路 (`road`)。
- **UI 同步**：地块编辑页下拉框同步移除“混合”，并修正了建筑和道路的内部标识符（统一为单数形式）。

### 2. 子类型组件一体化重构
- **交互升级**：将原有的“下拉框 + 独立管理面板”重构为**单一的一体式下拉输入组件**。
- **内嵌删除**：在下拉列表的每一项右侧嵌入了删除按钮，支持项内直接管理，点击主体区域触发选择，点击图标触发删除。
- **全类型应用**：原本仅农田支持的增强交互现已应用到所有 5 类地块。
- **大类隔离**：子类型列表严格按当前地块大类过滤，确保数据隔离。

### 3. 默认子类型配置接入
- **初始化脚本**：新增 Django 管理命令 `init_subcategories`，支持一键导入 60+ 个专业地块子类别（涵盖林、农、建、水、路）。
- **统计增强**：下拉项内保留了 `(区域内数量/全库数量)` 的动态统计展示。

### 4. 其它优化
- **UI 细节**：移除了属性面板中冗余的“备注”输入框（已集成至子类型）。
- **后端加固**：完善了子类别删除接口的业务检查，防止删除正在被使用的分类项。
- **工程规范**：更新了 `.gitignore`，规范化了 Git 提交信息。

## 本次更新概览 (2026-04-17)
本次更新深度优化了地形编辑器的视口控制逻辑与数据兼容性，显著提升了在加载大规模或异构坐标系地块时的交互稳定性。

### 主要变更点

#### 1. 视口控制逻辑闭环 (Viewport Management)
- **统一调度中心**：封装 `applyInitialViewportByMode` 作为视口切换的唯一入口，支持编辑、预览、列表多种模式。
- **智能优先级**：视口聚焦优先级重构为：**实际渲染地块边界 > `boundary_json` 弱兜底 > 地块中心点 > 重庆市全域边界**。
- **异步鲁棒性**：针对 Leaflet 渲染延迟，在 `loadAreaEditDetail` 中引入了多级重试聚焦机制，确保在异步地块加载完成后精准对齐。

#### 2. 坐标系与数据格式兼容
- **自动坐标转换**：内置 EPSG:3857 (Web Mercator) 到 EPSG:4326 (WGS84) 的自动识别与转换逻辑，完美支持存量 legacy 数据。
- **GeoJSON 规格化**：强化了对嵌套 JSON 字符串的递归解析（`normalizeGeoJSON`），提高了对后端多样化 API 响应的容错率。

#### 3. UI 交互与工程优化
- **界面精简**：从 [editor.html](file:///e:/PythonWeb/uav_platform/templates/terrain/editor.html) 中移除了冗余的 10m/1000m 参考网格开关，聚焦核心编辑体验。
- **代码规范化**：对编辑器模板进行了全量格式化，统一了缩进与属性排列规范，提升了代码可读性。
- **图层冲突修复**：修正了 `loadTerrainData` 中业务图层可能干扰编辑器聚焦的问题，实现了编辑模式下的图层行为隔离。

## 本次更新概览 (2026-04-18)
本次更新重点修复了地图编辑器中的底图切换失效问题，并彻底移除了历史地块的残留显示逻辑，提升了系统的稳定性与交互纯净度。

### 主要变更点

#### 1. 底图切换逻辑重构 (Base Map Management)
- **统一底图管理**：在 `TerrainEditor` 中建立了严格的底图注册与切换机制，支持卫星底图、标准底图、等高线底图的无缝切换。
- **防止图层叠加**：重写了 `switchBasemap` 函数，引入了暴力清理逻辑，确保在切换底图前彻底移除旧有 TileLayer 实例，解决“卫星图层残留覆盖”问题。
- **初始化链路优化**：修改了 `map_init.js` 的 `initMap` 函数，支持跳过默认加载，由编辑器统一控制首屏底图加载，消除了初始化时的图层竞争。
- **UI 状态同步**：左侧工作台“底图模式”卡片文字与地图实际底图实现实时联动。

#### 2. 历史地块辅助功能彻底移除
- **全局功能开关**：新增 `ENABLE_HISTORY_PLOT_ASSIST` 常量作为全局控制开关，默认设为 `false`。
- **拦截加载链路**：在 `loadTerrainData` 和 `toggleHistoryHints` 中增加了开关检查，彻底切断了历史地块模拟数据的请求与渲染。
- **主动清理残留**：在编辑器初始化阶段增加了 `cleanupHistoricalLayers` 逻辑，确保即使接口返回了冗余数据，也不会在地图上产生视觉残留。
