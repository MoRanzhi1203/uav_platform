# 地块创建与编辑工作台最终推荐方案

## 1. 需求重构与产品定义

**模块名称**：地块创建与编辑工作台

**定义**：一个专门用于创建和编辑地形地块的大型地图编辑界面，基于地形特征进行智能选区和精细编辑。

**与当前地形管理列表页的关系**：
- 作为地形管理模块的扩展功能
- 通过"新增地形"按钮进入
- 编辑完成后返回地形管理列表页

**为什么应该独立成一个大地图编辑界面**：
- 需要更大的地图编辑空间
- 集成了复杂的编辑工具和功能
- 提供专业的GIS编辑体验
- 避免在列表页中过度复杂

**与普通GIS浏览页的区别**：
- 提供专业的编辑工具集
- 支持基于地形特征的智能选区
- 支持精细的边界编辑
- 集成了业务数据和逻辑

## 2. 页面结构设计

**整体布局**：
- **顶部工具栏**：包含工具选择、操作按钮和标题
- **左侧图层/区域面板**：包含图层管理和已选地块列表
- **中间大地图编辑画布**：主要的编辑区域
- **右侧属性编辑面板**：包含地块属性编辑表单
- **底部状态栏**：显示坐标、面积、顶点数和编辑模式

**详细设计**：

1. **顶部工具栏**：
   - 页面标题：地块创建与编辑工作台
   - 工具组：
     - 选择工具：普通选择、智能选区、多选、合并
     - 编辑工具：顶点编辑、边编辑、裁剪、橡皮擦、画笔
     - 操作工具：撤销、重做、取消、保存

2. **左侧图层/区域面板**：
   - 图层管理：
     - 基础图层：地形边界、风险区域、禁飞区
     - 业务区域：林区边界、农田边界
   - 已选地块：显示当前选中的地块列表，支持单独删除
   - 合并按钮：合并选中的地块

3. **中间大地图编辑画布**：
   - 大尺寸地图区域
   - 支持缩放、平移等基础操作
   - 显示各类图层数据
   - 支持绘制和编辑多边形

4. **右侧属性编辑面板**：
   - 地块名称
   - 地块类型
   - 面积（自动计算）
   - 风险等级
   - 描述
   - 地理信息：中心点、边界点数量

5. **底部状态栏**：
   - 光标位置坐标
   - 选中区域面积
   - 顶点数量
   - 当前编辑模式

## 3. 地图编辑能力设计

**工具系统**：

1. **选择工具**：
   - 点击选择单个地块
   - 支持框选多个地块
   - 显示选中状态

2. **智能选区工具**：
   - 点击地图区域自动识别地块
   - 基于地形特征扩展选区
   - 自动创建选区多边形

3. **多选工具**：
   - 支持选择多个地块
   - 显示多选状态
   - 为合并操作做准备

4. **合并工具**：
   - 将多个选中的地块合并为一个
   - 优化合并后的边界
   - 自动计算合并后的面积

5. **顶点编辑工具**：
   - 显示多边形的顶点
   - 支持顶点拖动
   - 支持顶点的添加和删除

6. **边编辑工具**：
   - 显示多边形的边
   - 支持边线拖动
   - 支持在边上添加新顶点

7. **裁剪工具**：
   - 支持用直线裁剪多边形
   - 将一个多边形分割为两个
   - 保持几何完整性

8. **橡皮擦工具**：
   - 擦除多边形的部分边界
   - 调整多边形形状
   - 保持边界闭合

9. **画笔工具**：
   - 补画多边形边界
   - 扩展多边形区域
   - 平滑边界

10. **撤销/重做**：
    - 记录操作历史
    - 支持撤销和重做操作
    - 确保编辑的可恢复性

11. **保存/取消**：
    - 保存编辑后的地块数据
    - 取消编辑并返回列表页
    - 提示未保存的更改

## 4. 地块选区逻辑设计

**智能选区逻辑**：
1. **点击识别**：用户点击地图区域后，系统分析点击位置的地形特征
2. **特征匹配**：基于地形类型（山地、丘陵、山谷等）和业务区域（林区、农田）进行匹配
3. **自动扩展**：从点击位置向外扩展，识别连续的相同特征区域
4. **边界生成**：基于识别的区域生成初始边界
5. **边界优化**：使用凸包算法优化边界，确保几何正确性

**多选与合并逻辑**：
1. **多选**：用户可以通过多次点击或框选选择多个地块
2. **合并准备**：系统收集所有选中地块的边界点
3. **边界合并**：使用凸包算法合并所有边界点，生成新的边界
4. **属性合并**：合并地块的属性信息，如面积、风险等级等
5. **验证**：确保合并后的地块几何上正确，没有自交

**基于地形特征的选区**：
- **地形边界**：基于现有的地形边界数据进行选区
- **业务边界**：基于林区、农田等业务区域边界进行选区
- **风险区**：考虑风险区域的边界
- **禁飞区**：避开禁飞区
- **已有地块**：基于现有地块的边界进行选区

## 5. 边界精细编辑逻辑

**顶点编辑**：
- **顶点拖动**：支持顶点的自由拖动
- **顶点吸附**：顶点可以吸附到其他顶点或边线
- **顶点添加**：在边上点击添加新顶点
- **顶点删除**：右键点击删除顶点

**边线编辑**：
- **边线拖动**：支持整条边线的拖动
- **边线调整**：通过拖动边线调整多边形形状
- **边线平滑**：支持对边线进行平滑处理

**高级编辑**：
- **裁剪**：使用直线裁剪多边形
- **橡皮擦**：擦除多边形的部分边界
- **画笔**：补画多边形边界
- **防止自交**：自动检测并防止多边形自交
- **边界优化**：自动优化边界点，减少冗余顶点

**实现优先级**：
1. **顶点拖动**：最基础的编辑功能
2. **边线拖动**：次基础的编辑功能
3. **顶点添加/删除**：增强编辑能力
4. **吸附功能**：提高编辑精度
5. **裁剪工具**：支持复杂编辑
6. **橡皮擦工具**：支持精细调整
7. **画笔工具**：支持边界补画
8. **防止自交**：确保几何正确性

## 6. 地图特征数据组织方案

**数据加载**：
- **地形边界**：加载当前业务区的地形边界数据
- **风险区域**：加载风险区域数据，用于避开高风险区域
- **禁飞区**：加载禁飞区数据，用于避开禁飞区域
- **林区边界**：加载林区边界数据，用于基于林区特征选区
- **农田边界**：加载农田边界数据，用于基于农田特征选区
- **已有地块**：加载现有地块数据，用于参考和编辑

**数据格式**：
- 使用GeoJSON格式存储地理数据
- 支持多边形、多点等几何类型
- 包含必要的属性信息

**数据管理**：
- 按需加载数据，提高性能
- 支持数据的缓存和更新
- 确保数据的一致性和准确性

## 7. 与当前Django项目集成方式

**入口方式**：
- **推荐**：跳转到独立路由页
- **理由**：
  - 提供更大的编辑空间
  - 避免在原页面中过度复杂
  - 便于独立管理和优化
  - 提供更好的用户体验

**页面组织**：
- **Django模板**：使用Django模板系统渲染编辑器页面
- **Vue**：负责数据绑定和响应式更新
- **jQuery**：负责DOM操作和事件处理
- **地图库**：使用Leaflet进行地图渲染和交互

**集成细节**：
1. **路由配置**：在`terrain/urls.py`中添加编辑器页面路由
2. **视图函数**：在`terrain/views.py`中添加编辑器页面视图和API视图
3. **模型设计**：在`terrain/models.py`中完善地形模型，支持存储地理数据
4. **前端集成**：
   - 修改"新增地形"按钮，使其跳转到编辑器页面
   - 实现编辑器页面的JavaScript逻辑
   - 集成地图库和编辑工具
5. **API交互**：
   - 实现保存地块数据的API
   - 实现获取地形数据的API
   - 确保前后端数据传输的正确性

## 8. 技术实现方式

**前端技术**：
- **地图库**：Leaflet 1.9.4 + Leaflet Draw 1.0.4
- **UI框架**：Bootstrap 5
- **JavaScript**：原生JavaScript + Vue.js + jQuery
- **数据格式**：GeoJSON

**后端技术**：
- **框架**：Django
- **数据库**：MySQL
- **API**：RESTful API
- **数据存储**：JSONField存储地理坐标

**核心模块**：
1. **TerrainEditor**：主编辑器类，管理整体编辑流程
2. **SmartSelection**：智能选区模块，实现基于地形特征的选区
3. **BoundaryEditor**：边界编辑模块，实现精细的边界编辑
4. **MapDataManager**：数据管理模块，处理数据加载和保存

**关键算法**：
- **凸包算法**：用于优化合并后的边界
- **点-in-多边形**：用于判断点是否在多边形内
- **线段相交**：用于检测多边形自交
- **距离计算**：用于吸附功能
